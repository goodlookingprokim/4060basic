# 🧑‍💻 GitHub 기초 사용법 (1차시 수업)

## 🎯 학습 목표
- Git과 GitHub의 차이를 이해할 수 있다.
- GitHub 계정을 만들고 저장소(Repository)를 생성할 수 있다.
- 파일을 업로드하고 변경 사항을 관리할 수 있다.

---

## 📌 1. Git과 GitHub란 무엇인가?

### ✅ Git
- 파일의 변경 이력을 관리하는 **버전 관리 시스템**
- 예: "이 파일을 언제 수정했지?" → Git이 기록해 줌

### ✅ GitHub
- Git을 온라인에서 사용할 수 있는 **클라우드 저장소 서비스**
- 다른 사람과 협업 가능

👉 한 줄 정리  
**Git = 기록 도구 / GitHub = 공유 공간**

---

## 📌 2. GitHub 회원가입

### 📝 단계
1. https://github.com 접속
2. "Sign up" 클릭
3. 이메일, 비밀번호 입력
4. 사용자 이름 생성

👉 Tip  
- 사용자 이름은 프로젝트에 공개됨 → 신중하게 설정

---

## 📌 3. Repository(저장소) 만들기

### 📦 Repository란?
- 프로젝트를 저장하는 공간

### 🛠️ 생성 방법
1. 오른쪽 상단 "+" 클릭
2. "New repository" 선택
3. 이름 입력 (예: `my-first-project`)
4. "Public" 선택
5. "Create repository" 클릭

---

## 📌 4. 파일 업로드하기

### 📁 방법 1 (가장 쉬운 방법)
1. "Add file" 클릭
2. "Upload files" 선택
3. 파일 드래그
4. "Commit changes" 클릭

---

## 📌 5. Commit이란?

### 💡 개념
- 변경 사항을 저장하는 행동

👉 예시  
- "index.html 수정"
- "이미지 추가"

👉 한 줄 정리  
**Commit = 저장 + 기록**

---

## 📌 6. README 파일 만들기

### 📖 README란?
- 프로젝트 설명서

### ✏️ 작성 방법
1. "Add file" → "Create new file"
2. 파일 이름: `README.md`
3. 내용 작성
